## 🐛 Critical Fix

### 移除 migration 006 里重复的 ALTER TABLE

**问题**：v1.0.26 上线后 `bugpin.migelab.com` 502。

**根因**：
- `initSchema()` 里有兜底 `ALTER TABLE reports ADD COLUMN seq INTEGER`（已 try/catch）
- 但 `migration 006_add_report_sequence.sql` 第一行也是 `ALTER TABLE reports ADD COLUMN seq INTEGER`（没 IF NOT EXISTS，SQLite 不支持）
- initSchema 先跑 → 加成功；runMigrations 跑 006 → ALTER 因「duplicate column name」抛错 → 整个 server 启动失败 → docker 容器反复重启 → nginx 反代 502

**修复**：

`006_add_report_sequence.sql` 改为：
- ❌ 删掉 `ALTER TABLE reports ADD COLUMN seq INTEGER`（initSchema 已兜底）
- ✅ 保留 `UPDATE ... WHERE seq IS NULL` 回填（加 WHERE seq IS NULL 让它彻底幂等）
- ✅ 保留 `CREATE UNIQUE INDEX IF NOT EXISTS` 和 `CREATE INDEX IF NOT EXISTS`

migration 006 没有进入 `migrations` 表（因为 ALTER 抛错时 `INSERT INTO migrations` 还没跑），所以重新部署后会自动重跑新的 006 内容并完成回填。
