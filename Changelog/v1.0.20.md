# Changelog 1.0.20

报告列表 4 项体验改进（lula 2026-05-28）。

## ✨ Features

### 反馈人独立列

原本作为 title 列下方小字嵌套显示，现拆出为独立 TableCell；显示 `reporterName || reporterEmail || '-'`。

### 反馈模块独立列

原本作为 title 列内 outline badge 嵌套，现拆出为独立 TableCell（对应一级页面名）。title 列同步移除嵌套的 reporter 小字 + module badge，避免列与列重复展示。

### 状态列可直接修改

原本只读 StatusBadge，改为 shadcn Select 下拉。`onChange` 复用 `bulkUpdateMutation` 单元素分支直接 PATCH 后端。点击区 `stopPropagation` 避免触发整行的 `goToReportDetail`。

### 新增「已开发」状态（developed）

含义：开发完成但未部署。介于 `in_progress` 与 `resolved` 之间，用于团队内部区分「写完代码等部署」和「真正发布」两个状态。

- **i18n**：`dashboard.developed` 中文「已开发」/ 英文「Developed」
- **admin Reports.tsx**：StatusBadge labels + 状态过滤 Select + 批量操作 DropdownMenu + 行内编辑 Select 全部加 SelectItem
- **admin Dashboard.tsx**：StatusBadge labels 同步
- **server validate.ts**：`updateReport` / `bulkUpdateReports` 两处 z.enum 加 `'developed'`
- **server database.ts**：reports 表 CREATE TABLE 的 status CHECK 约束加 `'developed'`（fresh init 路径）

## 🗄️ Database Migration

**`migrations/005_extend_status_developed.sql`**：SQLite 不支持 ALTER CHECK，标准 `rename + recreate + copy + drop` 模式重建 reports 表。FTS5 触发器 / 索引由 `initSchema` 的 `CREATE IF NOT EXISTS` 兜底重建。`INSERT INTO reports_fts VALUES('rebuild')` 修 rowid mismatch。

## 🚀 Deployment Notes

- 状态新增 `developed` 不冲击历史数据（CHECK 是 SET 扩展，旧 4 值仍合法）
- 005 migration recreate reports 表会瞬间删 FTS rowid 然后 rebuild —— 生产数据量目前不大（几十条），应该没问题
