# Changelog 1.0.17

## 🚀 CI/CD

### 新增 GitLab CI 流水线（与 GitHub Actions 并行可用）

仓库新增 `.gitlab-ci.yml`，把现有 `.github/workflows/*.yml`（4 个 workflow：build/release/version-bump/deploy）等价适配为单个 GitLab CI 配置，便于在自托管 GitLab 实例（wendao-ai group）上独立跑构建、发版与部署。

**结构**：

- `stages: [build, post-build]`，4 个 job 通过 `needs` 组成 DAG，行为与 GHA workflow_run 链等价
- `build` → 构建 Docker 镜像并推到 `zot.migelab.com`，tag 同时打 `<version>` 与 `latest|dev`
- `version-bump` → 把 `package.json` / `src/server/package.json` / `src/admin/package.json` 版本同步到对应分支（dev / main）
- `release` → 仅生产 tag 触发，从 `Changelog/v<X.Y.Z>.md` 生成 GitLab Release
- `deploy` → 仅生产 tag 触发，SSH 到阿里云 ECS `docker pull` + 重启 bugpin 容器

**复用**：

- 走 wendao-ai group runner（tag=`docker`），不依赖 GitHub-hosted runner
- CI 变量与 GHA secrets 一一对应：`ZOT_USERNAME/ZOT_PASSWORD/ECS_HOST/ECS_USER/ECS_SSH_KEY`，新增 `GITLAB_PUSH_TOKEN` 用于 version-bump 回写
- workflow.rules 触发条件：`v*.*.*` tag 或 web 手动触发

**适配重点**：

- 不依赖 GitHub Actions 生态（`docker/login-action`/`appleboy/ssh-action`/`softprops/action-gh-release` 等），全部走 shell + 标准 docker/openssh
- 直接使用宿主 Docker daemon（runner 挂 `/var/run/docker.sock`），免 dind 开销
- Release 用 GitLab 原生 `release:` 关键字 + `release-cli` image
