# Changelog 1.0.26

ID 改为「项目缩写 + 自增数字」形式（如 `MIGE-7`），便于沟通引用（lula 2026-06-01）。

## ✨ Features

### 报告稳定短编号（per-project 自增）

之前 v1.0.25 用 `rpt_xxx` 前 8 位 hex 作为 ID，太长不好读。改成：

- **`<项目缩写>-<seq>`**，如 `MIGE-7`、`MIGE-8`
- **项目缩写**：projectName 去空格特殊字符后取前 8 位大写
- **seq**：per-project 自增整数，从 1 开始
- **点击 cell** 复制 `MIGE-7` 到剪贴板
- **hover** 显示完整 `rpt_xxx` ID
- **兜底**：老数据 seq=null 时回退到 8 位 hex

### 数据库改动

**`migrations/006_add_report_sequence.sql`**：

1. `ALTER TABLE reports ADD COLUMN seq INTEGER`
2. **回填**：每个 project 内按 `created_at, id` 排序，相关子查询给每行算 `COUNT(seq <= 自己)` 作为 seq
3. **UNIQUE 索引** `uk_reports_project_seq (project_id, seq)`：防并发 race / bug 重复发号
4. **普通索引** `idx_reports_project_seq (project_id, seq)`：按 project + seq 排序加速

### Create 路径并发安全

`reportsRepo.create()` 改为：

1. `BEGIN TRANSACTION`
2. `SELECT COALESCE(MAX(seq), 0) + 1 FROM reports WHERE project_id = ?`
3. `INSERT INTO reports (..., seq) VALUES (..., ?)`
4. `COMMIT` / 失败 `ROLLBACK`

UNIQUE 索引兜底防并发 race。
