# Changelog 1.0.22

修复 v1.0.21 指派功能在第二条及之后的报告报 400 的问题（lula 2026-05-28 反馈）。

## 🐛 Fixes

### 指派列错误处理 + 防御性校验

**问题**：v1.0.21 上线后，列表页指派列第一条改成功，后续报告点指派 → 后端 400 「更新报告失败」。

**改动**：

1. **`bulkUpdateMutation.onError` 把后端真实错误冒到 toast**
   原 `toast.error(t('reports.failedUpdate'))` → 改为 `getApiErrorMessage(error, default)`，定位具体失败原因（INVALID_ASSIGNEE / NOT_FOUND / ...）

2. **`handleAssigneeChange` 前端校验 sentinel**
   - `value === ASSIGN_TO_ME` 但 `user.id` 缺失时，提示失败并放弃（避免把 `'__assign_to_me__'` 当 user id 发到后端）
   - 任何 `__` 前缀的未识别 sentinel 直接 return，避免污染请求

**根因猜测**：v1.0.21 的 `handleAssigneeChange` 在 `user?.id` 为 falsy 时会落到 `else` 分支把 sentinel 字符串当成 user id 发出去，后端 `validateAssignee` 拒绝并返回 400。现在前端拦截 + 后端错误冒到 UI，下次复现就能精准定位。
