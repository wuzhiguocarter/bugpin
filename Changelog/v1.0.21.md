# Changelog 1.0.21

报告列表 3 项体验改进（lula 2026-05-28，BugPin 二轮迭代）。

## ✨ Features

### 反馈模块自动从 URL 推导

之前 `module` 字段全是 `-`，根因：项目未配 `moduleRules`，整列没数据。

现改为 `moduleRules` 都不命中时自动从 URL 推「一级页面名」：

- **优先级**：hash 路由（`#/foo/bar` → `foo`）→ pathname 第一段
- **跳过框架前缀**：`admin / app / main / index / pages / page` 等无意义段，继续取下一个真实页面名
- **URL decode**：支持中文路径段（如 `/样品管理/list` → `样品管理`）
- **服务端实现**：`reports.service.ts` 新增 `derivePageName(url)`，`createForProject` 在 moduleRules 不命中时调用

### 历史数据一次性回填

启动时 `backfillModuleFromUrl()` 扫描 `module IS NULL` 的 reports，按 `metadata.url` 推导出 module 写回。用 `migrations` 表里的 sentinel `_backfill_module_from_url_v1` 标记已执行，防止重启重跑。

### 报告列表加 ID 序号列

表头加 `#` 列（w-[60px]，`tabular-nums` 等宽），每行显示 `(page - 1) * limit + index + 1`，方便用户口语化引用「第 N 条」。

### 指派支持指派给我 + 列表页直接指派

原本指派列只读，改为内联 Select：

- **指派给我**（仅当当前用户 ≠ 已指派人时显示）：一键自指派
- **未分配**：撤销当前指派
- **所有可指派用户**：直接选
- 复用 `bulkUpdateMutation` 单元素分支 PATCH 后端
- 点击区 `stopPropagation` 避免触发整行的 `goToReportDetail`
- 只读用户（非 admin/editor）保持原 `AssigneeDisplay`

## 🌐 i18n
